# セットアップ手順

## 前提条件

- Flutter SDK 3.0以上がインストールされていること
- Dart 3.0以上がインストールされていること

## セットアップ手順

### 1. Flutterプロジェクトの初期化

このプロジェクトは既にFlutterプロジェクトとして構成されていますが、プラットフォーム固有の設定ファイル（Android/iOS/Web）が必要です。

以下のコマンドを実行して、プラットフォーム設定ファイルを生成してください：

# 「bash」はUnixやmacOS、Linuxで広く使われるシェル（コマンドラインインタプリター）です。
# 「```bash」で囲まれたコードは「bash（端末）」で実行できるコマンドを意味します。
# 
# Windowsでこのコマンド（flutter create .）をPowerShellやコマンドプロンプトで実行した場合、
# すでにFlutterプロジェクトのディレクトリ内であれば、「既存のプロジェクトに対して create は使えません」といった
# エラーになることがあります。
# 
# 原因:
# - flutter create . は新しくプロジェクトを作成するコマンドですが、すでにFlutterプロジェクトで
#   必要なファイル（pubspec.yamlなど）が存在すると上書きを避けるためにエラーになります。
# - 対策としては、「プラットフォームごとのフォルダ（android/ ios/ web/）」が消えているときなど、
#   明示的に再生成したい場合に使います。
# - 既にプロジェクトが正しく構成されていれば、このコマンドをスキップしても問題ありません。
flutter create .
```

このコマンドは既存のファイルを上書きしないため、安全に実行できます。

### 2. 依存関係のインストール

```bash
flutter pub get
```

### 3. アセットフォルダの作成

```bash
mkdir assets
```

（現在は空のフォルダで問題ありません）

### 4. アプリの実行

#### Android/iOS（実機またはエミュレータ）
```bash
flutter run
```

#### Web
```bash
flutter run -d chrome
```

## トラブルシューティング

### エラー: "No devices found"
- AndroidエミュレータまたはiOSシミュレータを起動してください
- または、実機を接続してください

### エラー: "PlatformException"
- `flutter clean`を実行してから、再度`flutter pub get`を実行してください

### エラー: "MissingPluginException"
- `flutter pub get`を再実行してください
- アプリを再起動してください
