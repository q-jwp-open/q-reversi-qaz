# Q-Reversi 設計書

## 1. プロジェクト概要

### 1.1 アプリケーション概要
Q-Reversiは、従来のリバーシゲームに量子コンピューティングの概念を組み合わせた革新的なパズルゲームです。プレイヤーは量子ゲートを使用して駒の状態を操作し、戦略的にボードを制御します。

### 1.2 技術スタック
- **本格開発フェーズ**: Flutter (iOS, Android, Flutter Web)
- **バックエンド**: Node.js + Express (必要に応じて)
- **データベース**: SQLite (モバイル), SharedPreferences (Flutter)

## 2. ゲーム仕様

### 2.1 基本ルール
- 8×8のボードを使用
- 白と黒とグレーの駒でゲーム進行
- 量子状態の駒（グレー駒、エンタングル状態）を導入
- 量子ゲートを使用して駒の状態を変更

### 2.2 初期状態
- **駒の配置**: すべての盤面にランダムで駒が割り当てられる
- **駒の種類**: 白、黒、グレープラス、グレーマイナスが均等な割合（各25%）
- **配置方法**: 8×8 = 64マスに、各種類16個ずつランダム配置

### 2.3 駒の種類（量子ビット）
1. **白石**: |0⟩ (量子ビットの基底状態)
2. **黒石**: |1⟩ (量子ビットの基底状態)
3. **グレープラス**: |+⟩ (重ね合わせ状態)
4. **グレーマイナス**: |-⟩ (重ね合わせ状態)
5. **エンタングル状態**:
   - ブラックホワイト: 上半分が黒、下半分が白
   - ホワイトブラック: 上半分が白、下半分が黒

### 2.4 ゲーム進行
1. **手番**: プレイヤーが交互に量子ゲートを作用させる（ターン制）
2. **ゲートを選択**: H / X / Y / Z / CNOT / SWAPを選択
3. **ゲートクールタイム**: それぞれのゲートにはクールタイムがあり、連続して使えない
   - H: 2ターン休み
   - X/Y: 6ターン休み
   - Z: 0ターン
   - CNOT: 3ターン休み
   - SWAP: 3ターン休み
4. **適用領域の選択**: 
 4-1. 1 bitゲート（H, X, Y, Z）の場合
    - 縦横は一列
      - 作用させたら、相手は同じ行/列に置けない
    - そのほか4マス四方にゲートを作用させられる
      - 作用させたら、相手は同じ4マスを選択できない
      - ただし、4マスのうちの2マスかぶりとかは許容。全く同じ4コマには置けない。ただし、直前に適用された4マスにかぶる部分は、ガードされて、ゲートが適用されない。
 4-2. 2 bitゲート（CNOT, SWAP）の場合
    - 縦横斜めに隣り合う2ビットに作用させられる
5. **エンタングルメント制限**: エンタングルされた駒にはゲート操作不可。

#### 操作UI
1. ボード外側に行/列選択ボタンを配置
2. ゲートタイプ別選択ルール
   - X/H/Y/Zゲート: 行/列ボタン使用可能、1マス選択で自動正方選択
   - SWAP/CNOTゲート: 行/列ボタン無効化、個別マス選択のみ。2マス続けて選択する
3. 自動正方選択
   - 基本、そのマス、及び右に1マス、下に1マス、右下に1マスの正方4マスを選択する
   - 右端/下端を選択した場合は自動補正し、そこを含む4マスの選択とする
4. UI改善
   - 選択された駒はハイライト
   - 2駒ゲートの場合は、1駒目と2駒目がわかるようにハイライト

### 2.5 ゲートクールタイムシステム
- **ターン制**: ゲームはターン制で進行
- **各ゲートのクールタイム**:
  - Hゲート: 2ターン休み（使用後2ターン使用不可）
  - Xゲート: 6ターン休み（使用後6ターン使用不可）
  - Yゲート: 6ターン休み（使用後6ターン使用不可）
  - Zゲート: 0ターン（クールタイムなし）
  - CNOTゲート: 3ターン休み（使用後3ターン使用不可）
  - SWAPゲート: 3ターン休み（使用後3ターン使用不可）
- **クールタイム管理**: 各プレイヤーごとに独立して管理され、自身のターンが進むごとに1ターンずつ減少

### 2.6 量子ゲート（反転作用）

#### 1駒ゲート
- **Xゲート**: 白と黒を入れ替える
- **Hゲート**: 
  - グレープラスと白を入れ替える
  - グレーマイナスと黒を入れ替える
- **Yゲート**: 
  - 白と黒を入れ替える
  - グレープラスとグレーマイナスを入れ替える
- **Zゲート**: グレープラスとグレーマイナスを入れ替える

#### 2駒ゲート
- **SWAPゲート**: 選んだ2駒を入れ替える
- **CNOTゲート**: 制御NOTゲート
  - 一つ目の駒の色に従って二つ目の駒の作用が変わる
  - 一つ目が自分の手番の色の場合、二つ目の駒にXを作用させる
  - 一つ目がグレー駒の場合、以下のルールに従って駒が切り替わる：

**CNOTゲート詳細ルール**:
| 1駒目 / 2駒目 | →CNOT→ | 1駒目/2駒目 |
|---|---|---|
| グレープラス/グレープラス | → | グレープラス/グレープラス |
| グレープラス/グレーマイナス | → | グレーマイナス/グレーマイナス |
| グレーマイナス/グレープラス | → | グレーマイナス/グレープラス |
| グレーマイナス/グレーマイナス | → | グレープラス/グレーマイナス |
| グレープラス/自分の手番の駒(白 or 黒) | → | ホワイトブラック/ブラックホワイト |
| グレープラス/相手の手番の駒(白 or 黒) | → | ホワイトブラック/ホワイトブラック |
| グレーマイナス/自分の手番の駒(白 or 黒) | → | ブラックホワイト/ホワイトブラック |
| グレーマイナス/相手の手番の駒(白 or 黒) | → | ブラックホワイト/ブラックホワイト |

### 2.7 禁止領域システム
- **1 bitゲート適用後の禁止領域**:
  - **行/列選択**: ゲートを作用させた行/列は、次の相手のターンで同じ行/列に置けない
  - **4マス選択**: ゲートを作用させた4マスは、次の相手のターンで全く同じ4マスを選択できない
    - ただし、4マスのうちの2マスが重複する場合は許容される
    - **補足**: 直前に適用された4マスにかぶる部分がある場合、そのかぶる部分にはゲートが適用されない（ガードされる）
- **禁止領域の管理**: 各プレイヤーごとに最後に適用した領域を記録し、相手プレイヤーの選択を制限
- **禁止領域の有効期間**: 相手の次のターンのみ有効（そのターンが終了すると解除される）

### 2.8 エンタングルメント制限
- **特別感の演出**: エンタングルされた駒は視覚的に特別な表示
- **操作制限**: エンタングルされた駒にはゲート操作不可。
    縦横1列にゲートを適用した場合、エンタングルした駒でゲート適用範囲は止まる。
    4マス四方範囲にゲートを適用した中にエンタングルした駒が存在した場合、エンタングルされた駒のみゲートが適用されない。
- **戦略的要素**: エンタングルメントを活用した防御戦略

### 2.9 ゲーム終了条件
1. **ターン制限**: 決められたターン数に達した場合
2. **測定操作**: ゲーム終了時に全駒に対して測定操作を実行

### 2.10 測定操作（ゲーム終了時）
- **白/黒**: そのまま（変化なし）
- **グレープラス/グレーマイナス**: 50%の確率で白か黒に反転
- **ブラックホワイト/ホワイトブラック**: 50%の確率で白か黒に反転
- **エンタングル状態**: 1駒目の測定結果に応じて2駒目が確定

### 2.11 ゲームモード
1. **チャレンジモード**: 特定の量子状態を作るミッション
2. **VSモード**: 2人対戦のモード。
3. **フリーランモード**: 一人で、盤面に対して自由にゲートを演算可能なモード。
4. **スタディモード**: 量子コンピューティングの理論を視覚的に学習する教育モード
5. **プロフェッショナルモード**: エンタングルメントにもゲートを作用させる高度なモード（将来拡張）

### 2.12 チャレンジモード詳細

#### 2.12.1 チャレンジシステムの特徴

**段階的学習**
- 各レベルで1つの概念に集中
- 前のレベルで学んだ内容を次のレベルで応用
- ゲームの基本操作、ゲートの作用を一つ一つパズル形式で段階的に学習

**戦略的示唆**
- エンタングルメントの戦略的価値を段階的に理解
- 「操作できないが有利になる」という逆説的な概念を学習
- 測定時の結果予測の重要性を理解

**報酬システム**
- レベルクリアで★獲得
- 手数に応じて★を与える。最短クリアで★3つ
- 30レベルごとに、1ステージとする。ステージ内のすべてのレベルが★3つの場合、バッジを獲得

**難易度調整**
- 初期状態の調整
- 使用可能ゲートの制限

#### 2.12.2 パズル要素
- **盤面状態の作成**: 特定の盤面状態を作るミッション
- **エンタングルメント**: 2つの駒をエンタングル状態にする
- **量子測定**: グレー駒を測定して確定状態にする

#### 2.12.3 パズル初期状態のソース
- q-reversi_challange-mode.csvを読んで、初期状態を決定する。
- それぞれ、下記ルールに従うものとする。
- level：レベル。1から順に増えていく。
- turn：最短クリア可能ターン
- available gate：使用可能ゲート
- victory condition：勝利条件

#### 2.12.4 CSVファイルフォーマット仕様
- **ファイル形式**: CSV（カンマ区切り）
- **エンコーディング**: UTF-8
- **構造**:
  - 1行目: ヘッダー行（1 bit gate, 2 bit gate等）
  - 2行目: ゲートタイプ定義行
  - 3行目: 空行
  - 4行目以降: レベルデータ
- **レベルデータ構造**:
  - level: レベル番号（整数）
  - turn: 最短クリア可能ターン数（整数）
  - available gate: 使用可能ゲート（カンマ区切り、例: "X, H, Z"）
  - victory condition: 勝利条件（文字列、例: "All white", "All black"）
  - 盤面データ: 8×8のマス目に駒の種類を記述
    - W: 白
    - B: 黒
    - +: グレープラス
    - -: グレーマイナス
    - BW: ブラックホワイト
    - WB: ホワイトブラック
  - comment: コメント（学習目標等）
- **エラーハンドリング**: 不正なフォーマットの場合はデフォルトレベルを読み込む

### 2.13 VSモード詳細

**対戦モード選択**:
- **対人戦**: 2人のプレイヤーが交互に操作
- **対CPU戦**: プレイヤー vs CPU（AI）

**盤面構成**:
- **8×8の盤面**: 64マスを使用
- **初期状態**: すべての盤面にランダムで駒が割り当てられる
  - 白、黒、グレープラス、グレーマイナスが均等な割合（各25%）

**プレイヤー設定**:
- **プレイヤー1**: 先攻（白を担当）
- **プレイヤー2**: 後攻（黒を担当）
  - 対人戦の場合: 2人目のプレイヤー
  - 対CPU戦の場合: CPU（AI）

**ターン制限**:
- **ゲーム全体の最大ターン数**を設定可能（デフォルト: 20ターン）
- 20/30/40/50/60ターンから選択可能
- ターン数が上限に達した場合、ゲーム終了し、測定を行う
- **注意**: ターン数は両プレイヤーの合計ターン数（例: 20ターン = プレイヤー1が10ターン + プレイヤー2が10ターン）

**勝利条件**: 
- 測定後、自分の色の駒が多い方が勝利
- 同数の場合は引き分け

**手番の決定**: 
- ランダムまたは設定で決定
- 対CPU戦の場合、プレイヤーが先攻か後攻かを選択可能

**クールタイム表示**:
- **自分のクールタイム**: ゲート選択パネルに表示（各ゲートボタンに残りターン数を表示）
- **相手のクールタイム**: 画面の上部またはサイドに小さく表示
  - 相手プレイヤーの各ゲートのクールタイム残りターン数を表示
  - アイコンと数値で表示（例: "X: 3" はXゲートが3ターン後に使用可能）
  - クールタイム中のゲートはグレーアウト表示

**ゲーム保存**: 
- 途中でゲームを保存・再開可能
- 対人戦・対CPU戦のどちらでも保存可能

**対CPU戦のAI仕様**:
- **難易度設定**: 初級、中級、上級から選択可能
- **AIの思考**:
  - 初級: ランダムにゲートを選択
  - 中級: 基本的な戦略（有利な位置にゲートを適用）
  - 上級: 高度な戦略（エンタングルメントの活用、禁止領域の戦略的使用）
- **AIのターン**: 自動的にゲートを選択・適用（アニメーション表示）

### 2.14 フリーランモード詳細
- **目的**: 量子ゲートの動作を学習・実験するためのモード
- **盤面構成**:
  - **8×8の盤面**: 64マスを使用（通常モードと同じ）
- **機能**:
  - 任意の初期状態を設定可能
  - すべてのゲートを無制限に使用可能
  - クールタイムなし
  - ターン制限なし
  - 測定操作を任意のタイミングで実行可能
- **制限事項**:
  - エンタングルメント制限は適用される
  - 禁止領域システムは適用されない
- **リセット機能**: 初期状態に戻す機能
  - 基本的には、ランダムな配置の初期状態に戻す
  - ただ、すべて白/黒/グレープラス/グレーマイナスに変更することもできる

### 2.15 スタディモード詳細

#### 2.15.1 スタディモードの目的
量子コンピューティングの理論的概念を視覚的・インタラクティブに学習するための教育モード。ブロッホ球による状態可視化、2ビットゲートの動作理解、量子アルゴリズムの実装体験を提供する。

#### 2.15.2 スタディ1: 1ビットゲート学習モード

**目的**: 単一量子ビットに対するゲート操作をブロッホ球で可視化して学習

**機能**:
- **ブロッホ球表示**: 選択したマス（量子ビット）の状態を3Dブロッホ球で表示
  - 球面上の点で量子状態を表現
  - リアルタイムで状態変化をアニメーション表示
- **ゲート適用の可視化**: 
  - ゲート選択時に、ブロッホ球上での作用を予測表示
  - ゲート適用後、実際の状態変化をアニメーション表示
- **状態情報表示**:
  - 現在の量子状態（|0⟩, |1⟩, |+⟩, |-⟩等）
  - ブロッホ球座標（θ, φ）
  - 確率振幅（α, β）
- **インタラクティブ操作**:
  - 1つのマスを選択してゲートを適用
  - ゲート適用前後の状態を比較表示
  - ゲートの連続適用による状態変化を追跡

**対応ゲート**:
- Xゲート: ブロッホ球のX軸周り180度回転
- Yゲート: ブロッホ球のY軸周り180度回転
- Zゲート: ブロッホ球のZ軸周り180度回転
- Hゲート: ブロッホ球の特定軸周り90度回転

**UI構成**:
- 左側: 1マスボード
- 右側: ブロッホ球表示エリア
  - 3D回転可能なブロッホ球
  - 状態ベクトル表示
  - ゲート選択パネル
  - 状態履歴表示
- 下側: ゲート選択＆適用エリア

**学習コンテンツ**:
- 各ゲートの説明テキスト
- ゲート適用のステップバイステップガイド
- 状態変化の数式表示（オプション）

#### 2.15.3 スタディ2: 2ビットゲート学習モード

**目的**: 2ビットゲート（CNOT, SWAP）の動作を視覚的に理解

**機能**:
- **2ビット状態の可視化**:
  - 選択した2つのマスの状態を同時表示
  - エンタングルメント状態の可視化
  - ベルの不等式などの概念の説明
- **ゲート適用の可視化**:
  - CNOTゲート: 制御ビットとターゲットビットの関係を明示
  - SWAPゲート: 2ビットの交換をアニメーション表示
- **状態テーブル表示**:
  - 2ビットの全状態（|00⟩, |01⟩, |10⟩, |11⟩）の確率振幅
  - エンタングルメントの度合い（エンタングルメントエントロピー）
- **インタラクティブ操作**:
  - 2つのマスを選択してゲートを適用
  - エンタングルメント生成の過程を可視化
  - 測定操作による状態収束の表示

**対応ゲート**:
- CNOTゲート: 制御NOTゲートの動作
- SWAPゲート: 2ビットの交換

**UI構成**:
- 上部: 1×2ボード（2マス選択可能）
- 中央: 2ビット状態表示エリア
  - 状態ベクトル表示
  - エンタングルメント可視化
  - 確率分布グラフ
- 下部: ゲート選択パネル、操作履歴

**学習コンテンツ**:
- エンタングルメントの概念説明
- CNOTゲートの真理値表
- ベルの不等式の実験（オプション）

#### 2.15.4 スタディ3: 量子アルゴリズム実装モード

**目的**: 1×4盤面（4マス）を用いてグローバーのアルゴリズムを模擬し、ガイド付きで学習

**盤面構成**:
- **マス数**: 1×4 = 4マス
- **状態数**: 2^4 = 16通り（各マスが白|0⟩または黒|1⟩の2状態）
- **状態表現**: |0000⟩, |0001⟩, |0010⟩, ..., |1111⟩（16通り）

**学習モード**:
- **ガイドモード**: ステップバイステップのガイドに従って操作（初心者向け）
- **練習モード**: ガイドなしで自由に操作（中級者向け）

**機能**:
- **オラクルゲート**:
  - 特定の条件を満たす状態をマークする特殊ゲート
  - システムが自動的に条件を設定（例: |1010⟩を探す、または白が2個の状態を探す等）
  - オラクル適用時に条件を満たす状態をハイライト表示
- **量子アルゴリズムの実装**:
  - **グローバーのアルゴリズム**: 未ソートデータベースの検索
- **ガイドシステム**:
  - 各ステップでの操作指示を表示
  - 次の操作をハイライト表示
  - 間違った操作時のフィードバック
  - ステップのスキップ機能（理解している場合）
- **状態確率表示**:
  - 右側パネルに16通りの状態すべての確率を表示
  - 各状態の確率を棒グラフで可視化
  - 条件を満たす状態の確率を強調表示

**実装可能なアルゴリズム**:

**グローバーのアルゴリズム（ガイド付き実装）**:

**ステップ1: 初期化フェーズ（全状態の重ね合わせを作る）**

**理論的背景**:
- グローバーアルゴリズムでは、まずすべての可能な状態を均等な重ね合わせ状態にする必要があります
- 4マスすべてを|+⟩状態にすることで、16通りの状態すべてが等しい確率（1/16 = 6.25%）で存在する状態になります
- これは「すべての可能性を同時に探索する」という量子計算の特徴です

**ガイド表示**:
- 「まず、すべてのマスを重ね合わせ状態にします」
- 「Hゲートを使って、各マスを|+⟩状態にしましょう」
- 「これにより、16通りの状態すべてが等しい確率で存在するようになります」

**操作手順**:
1. ガイドが「各マスにHゲートを適用してください」と表示
2. ユーザーは1列一括適用ボタンで一括実行
3. Hゲートボタンをクリック
4. 各マスが|+⟩（グレープラス）に変化
5. 右側の確率分布パネルで、16通りの状態すべてが6.25%になっていることを確認

**視覚的フィードバック**:
- 適用済みのマスは緑色の枠で表示
- 現在のステップのマスは黄色でハイライト
- 右側の確率分布グラフが更新（全16状態が均一な確率6.25%）

**ステップ2: オラクル条件の自動設定**

**オラクルとは**:
- **オラクル（Oracle）**は、量子アルゴリズムにおいて「解を識別する関数」を表します
- オラクルは、条件を満たす状態（解）に対して「印をつける」役割を果たします
- グローバーアルゴリズムでは、オラクルが「探したい状態」を識別します
- オラクルは「ブラックボックス」として扱われ、内部の実装は気にせず、条件を満たすかどうかだけを判定します

**システムによる自動設定**:
- システムが自動的に探す状態を決定（例: |1010⟩、または白が2個の状態等）
- ユーザーは条件を設定する必要はありません
- 条件を満たす状態が1つまたは複数自動的に選択されます

**ガイド表示**:
- 「システムが自動的に探す状態を設定しました」
- 「オラクルは、条件を満たす状態に印をつける関数です」
- 「今回は、状態|○○○○⟩を探します」（実際の状態を表示）

**操作手順**:
1. ガイドが「オラクル条件が設定されました」と表示
2. 条件を満たす状態が右側の確率分布パネルで赤色でハイライト表示
3. ガイドが「条件を満たす状態が○個見つかりました」と表示

**視覚的フィードバック**:
- 条件を満たす状態: 右側の確率分布パネルで赤色でハイライト
- 条件を満たさない状態: 通常表示
- マッチング数がカウント表示（例: 「1個の状態が条件を満たします」）

**ステップ3: オラクルゲートの適用（位相反転）**

**オラクルゲートの動作**:
- オラクルゲートは、条件を満たす状態に対して「位相反転」を行います
- 位相反転とは、状態の確率振幅に-1を掛ける操作です
- 数学的には: O|x⟩ = -|x⟩ (条件を満たす場合), O|x⟩ = |x⟩ (条件を満たさない場合)
- この操作により、条件を満たす状態の確率振幅の符号が反転します
- 確率は |振幅|² で計算されるため、符号の反転だけでは確率は変わりませんが、次の拡散演算子との組み合わせで効果が現れます

**ガイド表示**:
- 「オラクルゲートを適用して、条件を満たす状態に位相反転を加えます」
- 「位相反転により、条件を満たす状態の確率振幅の符号が反転します」
- 「確率分布パネルで、条件を満たす状態の確率振幅が変化することを確認してください」

**操作手順**:
1. ガイドが「オラクルゲートを適用します」と表示
2. 「オラクルゲート適用」ボタンが有効化
3. ボタンをクリック
4. システムが自動的に条件を満たす状態に位相反転を適用
5. 右側の確率分布パネルが更新（条件を満たす状態の確率振幅が反転）

**視覚的フィードバック**:
- 位相反転のアニメーション（波のような効果）
- 右側の確率分布パネルで、条件を満たす状態の確率振幅が反転することを表示
- 数式表示: 「O|x⟩ = -|x⟩ (if condition), |x⟩ (otherwise)」
- 確率は変わらないが、振幅の符号が変わることを説明

**ステップ4: 拡散演算子の適用（振幅増幅）**

**拡散演算子とは**:
- **拡散演算子（Diffusion Operator）**は、平均値周りの反転を行う演算子です
- 数学的には: D = 2|s⟩⟨s| - I
  - |s⟩は全状態の均等な重ね合わせ状態（初期状態）
  - この演算子は、各状態の確率振幅から平均値を引いて、符号を反転させます
- 拡散演算子の効果:
  - 平均より上にある確率振幅: さらに増加
  - 平均より下にある確率振幅: さらに減少
- オラクルゲートで位相反転された状態（平均より下）は、拡散演算子により平均より上に移動し、確率が増加します
- これが「振幅増幅」の仕組みです

**拡散演算子の実装**:
- 各マスにHゲートを適用して重ね合わせ状態に戻す
- 平均値周りの反転を計算: 各状態の確率振幅から平均値を引き、符号を反転
- 条件を満たす状態の確率振幅が増幅される

**ガイド表示**:
- 「次に、拡散演算子を適用して解の確率を増やします」
- 「拡散演算子は平均値周りの反転を行い、条件を満たす状態の確率を増やします」
- 「右側の確率分布パネルで、条件を満たす状態の確率が増加することを確認してください」

**操作手順**:
1. ガイドが「拡散演算子を適用します」と表示
2. 「拡散演算子適用」ボタンが有効化
3. ボタンをクリック
4. システムが自動的に以下を実行:
   - 各マスにHゲートを適用（重ね合わせ状態に戻す）
   - 平均値周りの反転を計算
   - 条件を満たす状態の確率振幅を増幅
5. 右側の確率分布パネルが更新（条件を満たす状態の確率が増加）

**視覚的フィードバック**:
- 全マスに波のようなアニメーション
- 右側の確率分布パネルで、条件を満たす状態の確率が上がることを表示
- 成功確率の数値表示（例: 「成功確率: 15%」）
- 数式表示: 「D = 2|s⟩⟨s| - I」

**ステップ5: 繰り返し（オラクル + 拡散演算子）**

**繰り返しの理由**:
- オラクルゲートと拡散演算子を繰り返し適用することで、条件を満たす状態の確率を段階的に増やします
- 1回の適用では確率が十分に上がらないため、最適な回数まで繰り返します
- 最適な繰り返し回数: k ≈ π/4 × √N
  - N=16（4マスの場合）: k ≈ π/4 × 4 ≈ 3回
  - これにより、条件を満たす状態の確率が最大（約94%）になります

**ガイド表示**:
- 「オラクルと拡散演算子を繰り返し適用します」
- 「最適な繰り返し回数は約3回です（4マスの場合）」
- 「各繰り返しで、条件を満たす状態の確率が増加します」

**操作手順**:
1. ガイドが「ステップ3と4を繰り返します」と表示
2. 繰り返しカウンターが表示（現在: 1回目/3回）
3. ユーザーが「次へ」ボタンをクリック
4. オラクルゲート適用（ステップ3）
5. 拡散演算子適用（ステップ4）
6. 成功確率が更新（例: 「成功確率: 25%」）
7. 右側の確率分布パネルで、条件を満たす状態の確率が増加することを確認
8. 繰り返しカウンターが更新（2回目/3回）
9. 3回まで繰り返し
10. 各繰り返しで成功確率が増加することを確認

**ステップ7: 古典手法と量子手法の比較（まとめ）**

**ガイド表示**:
- 「グローバーアルゴリズムの優位性を理解しましょう」
- 「古典計算と量子計算の違いを確認します」

**古典手法での探索**:
- **計算量**: O(N) - 線形時間
- **16状態の場合**:
  - **最悪の場合**: 16回の試行が必要（最後まで調べる必要がある）
  - **平均的な場合**: 約8回の試行が必要（16÷2）
  - **最良の場合**: 1回の試行で見つかる（最初に当たる場合）
- **手法**: 各状態を1つずつ順番に調べていく
- **例**: 16個の箱から1つの当たりを探す場合、最悪16回、平均8回の開封が必要

**量子手法（グローバーアルゴリズム）での探索**:
- **計算量**: O(√N) - 平方根時間
- **16状態の場合**:
  - **必要な繰り返し回数**: 約3回（π/4 × √16 ≈ 3）
  - **成功確率**: 約94%（最適回数で）
  - **各繰り返しでの操作**: オラクルゲート + 拡散演算子（2つの操作）
  - **総操作数**: 約6回（初期化1回 + 繰り返し3回 × 2操作）
- **手法**: すべての状態を同時に重ね合わせ状態で探索し、解の確率を増幅
- **例**: 16個の箱を同時に開けて、当たりの確率を段階的に増やしていく

**比較表**:

| 手法 | 計算量 | 16状態の場合 | 特徴 |
|------|--------|--------------|------|
| **古典手法** | O(N) | 最悪16回、平均8回 | 1つずつ順番に調べる |
| **量子手法** | O(√N) | 約3回の繰り返し（約6操作） | すべての状態を同時に探索 |

**量子優位性**:
- **16状態の場合**: 古典手法では最悪16回、量子手法では約3回の繰り返し
- **状態数が増えるほど差が大きくなる**:
  - 100状態: 古典最悪100回 vs 量子約8回
  - 10,000状態: 古典最悪10,000回 vs 量子約79回
  - 1,000,000状態: 古典最悪1,000,000回 vs 量子約785回
- **量子計算の利点**: 状態数が大きくなるほど、古典計算との差が劇的に大きくなる

**まとめメッセージ**:
- 「今回の16状態の探索では、古典手法では最悪16回、平均8回の試行が必要でした」
- 「一方、グローバーアルゴリズムでは、約3回の繰り返し（約6操作）で約94%の確率で解を見つけることができました」
- 「これが量子計算の優位性です。状態数が大きくなるほど、その差は劇的に大きくなります」

#### 3.4.3.1 AI思考アルゴリズム（対CPU戦）

**AIの思考プロセス**:

1. **使用可能なゲートの評価**:
   - クールタイムが終了しているゲートをリストアップ
   - 各ゲートの戦略的価値を評価

2. **盤面状態の評価**:
   - 現在の盤面状態を分析
   - 自分の色の駒の数、位置を評価
   - エンタングル状態の有無を確認
   - 禁止領域の影響を考慮

3. **ゲート選択アルゴリズム（難易度別）**:

   **初級（ランダム）**:
   - 使用可能なゲートからランダムに選択
   - 適用領域もランダムに選択

   **中級（基本的な戦略）**:
   - 自分の色の駒が多くなる選択肢を優先
   - 一定ターン後（50%以降）に、エンタングル状態を作成して防御
     - ターン数が50%を超えたら、エンタングル評価を上げる（0.3 → 0.5）
     - CNOTゲートを使用してエンタングル状態を作成する選択肢を優先
   - 禁止領域を活用して相手の選択肢を制限
   - クールタイムの短いゲート（Zゲート等）を優先

   **上級（高度な戦略）**:
   - 測定後の予測確率を計算
   - **エンタングルメントを戦略的に活用**:
     - ターン数に応じて段階的にエンタングル評価を上げる
       - 30%以降: エンタングル評価 0.5
       - 50%以降: エンタングル評価 0.7
       - 70%以降: エンタングル評価 1.0（最大）
     - 重要な領域（自分の駒が多い行/列、中央付近）をエンタングルで防御
     - エンタングルペアを戦略的に配置して、相手のゲート適用を阻止
     - 測定時の期待値を考慮したエンタングル配置
   - 禁止領域を最大限に活用
   - 相手のクールタイムを考慮した戦略
   - 長期的な戦略を考慮

4. **適用領域の選択**:
   - 選択したゲートに応じて適用領域を決定
   - 1ビットゲート: 行/列または4マス選択
   - 2ビットゲート: 隣接する2マスを選択

5. **操作の実行**:
   - 選択したゲートと領域で操作を実行
   - アニメーション表示

**実装上の考慮事項**:
- AIの思考時間を適切に設定（0.5秒～2秒）
- 思考中の表示をユーザーに提示
- 難易度に応じて思考の深さを調整

```dart
  Future<AIAction> think(GameState gameState) async {
    // 思考時間のシミュレーション（0.5秒～2秒）
    await _simulateThinkingTime();
    
    // 使用可能なゲートを取得
    final availableGates = _getAvailableGates(gameState);
    if (availableGates.isEmpty) {
      throw Exception('No available gates');
    }
    
    // 難易度別の思考
    switch (difficulty) {
      case AIDifficulty.beginner:
        return _thinkBeginner(gameState, availableGates);
      case AIDifficulty.intermediate:
        return _thinkIntermediate(gameState, availableGates);
      case AIDifficulty.advanced:
        return _thinkAdvanced(gameState, availableGates);
    }
  }
  
  // 思考時間のシミュレーション
  Future<void> _simulateThinkingTime() async {
    final minTime = 500;  // 0.5秒
    final maxTime = 2000;  // 2秒
    final thinkTime = minTime + random.nextInt(maxTime - minTime);
    await Future.delayed(Duration(milliseconds: thinkTime));
  }
}
```

```dart
AIAction _thinkIntermediate(GameState gameState, List<GateType> availableGates) {
  // 各ゲートと適用領域の組み合わせを評価
  final candidates = <AIAction>[];
  
  for (final gate in availableGates) {
    final targets = _getPossibleTargets(gameState, gate);
    
    for (final target in targets) {
      // 仮想的にゲートを適用
      final simulatedState = _simulateGateApplication(
        gameState,
        gate,
        target,
      );
      
      // 盤面を評価（難易度を考慮）
      final evaluation = _evaluateBoard(simulatedState, myColor, difficulty);
      
      candidates.add(AIAction(
        gate: gate,
        target: target.target,
        positions: target.positions,
        score: evaluation.totalScore,
      ));
    }
  }
  
  // 評価スコアが高い順にソート
  candidates.sort((a, b) => b.score.compareTo(a.score));
  
  // 上位3つからランダムに選択（完全に最適解を選ばない）
  final topCandidates = candidates.take(3).toList();
  return topCandidates[random.nextInt(topCandidates.length)];
}

// 盤面評価関数（難易度を考慮）
BoardEvaluation _evaluateBoard(
  GameState gameState,
  PlayerColor myColor,
  AIDifficulty difficulty,
) {
  final board = gameState.board;
  var myPieceScore = 0.0;
  var opponentPieceScore = 0.0;
  var entanglementScore = 0.0;
  var forbiddenAreaScore = 0.0;
  
  // 各マスを評価
  for (int row = 0; row < 8; row++) {
    for (int col = 0; col < 8; col++) {
      final piece = board.getPiece(row, col);
      final position = Position(row, col);
      
      // 自分の色の駒を評価
      if (_isMyColor(piece, myColor)) {
        myPieceScore += _getPieceValue(piece, position);
      }
      // 相手の色の駒を評価
      else if (_isOpponentColor(piece, myColor)) {
        opponentPieceScore += _getPieceValue(piece, position);
      }
      
      // エンタングル状態の評価（難易度とターン数を考慮）
      if (piece.isEntangled) {
        entanglementScore += _getEntanglementValue(
          piece,
          myColor,
          gameState,
          difficulty,  // 難易度を渡す
        );
      }
    }
  }
  
  // 禁止領域の評価（相手の選択肢を制限している）
  forbiddenAreaScore = _evaluateForbiddenAreas(gameState, myColor);
  
  // 総合評価（難易度に応じて重みを調整）
  double entanglementWeight = 0.5;
  double forbiddenAreaWeight = 0.3;
  
  // 上級AIではエンタングルと禁止領域をより重視
  if (difficulty == AIDifficulty.advanced) {
    entanglementWeight = 0.8;  // エンタングルをより重視
    forbiddenAreaWeight = 0.5;  // 禁止領域もより重視
  } else if (difficulty == AIDifficulty.intermediate) {
    entanglementWeight = 0.6;  // 中級でもエンタングルをやや重視
  }
  
  final totalScore = myPieceScore - opponentPieceScore * 0.8 
      + entanglementScore * entanglementWeight 
      + forbiddenAreaScore * forbiddenAreaWeight;
  
  return BoardEvaluation(
    myPieceScore: myPieceScore,
    opponentPieceScore: opponentPieceScore,
    entanglementScore: entanglementScore,
    forbiddenAreaScore: forbiddenAreaScore,
    totalScore: totalScore,
  );
}

// 駒の価値評価
double _getPieceValue(Piece piece, Position position) {
  double value = 1.0;
  
  // 駒の種類による価値
  if (piece.type == PieceType.white || piece.type == PieceType.black) {
    value = 1.0;  // 確定状態
  } else if (piece.type == PieceType.grayPlus || 
             piece.type == PieceType.grayMinus) {
    value = 0.5;  // 重ね合わせ状態（測定時に50%で確定）
  }
  
  // 位置による価値（中央が高い）
  final centerDistance = _getDistanceFromCenter(position);
  value *= (1.0 - centerDistance * 0.1);
  
  return value;
}

// エンタングル状態の価値評価（改善版）
double _getEntanglementValue(
  Piece piece, 
  PlayerColor myColor,
  GameState gameState,
  AIDifficulty difficulty,
) {
  if (!piece.isEntangled || !_isMyColor(piece, myColor)) {
    return 0.0;
  }
  
  final turnCount = gameState.turnCount;
  final maxTurns = gameState.maxTurns;
  final turnRatio = turnCount / maxTurns;  // ゲームの進行度（0.0-1.0）
  
  // 基本防御価値
  double baseValue = 0.3;
  
  // 難易度とターン数に応じた評価の調整
  switch (difficulty) {
    case AIDifficulty.beginner:
      // 初級: エンタングルを考慮しない
      return 0.0;
      
    case AIDifficulty.intermediate:
      // 中級: 一定ターン後（50%以降）にエンタングル評価を上げる
      if (turnRatio >= 0.5) {
        baseValue = 0.5;  // 後半戦でエンタングルを重視
      }
      break;
      
    case AIDifficulty.advanced:
      // 上級: ターン数に応じて段階的にエンタングル評価を上げる
      if (turnRatio >= 0.7) {
        baseValue = 1.0;  // 終盤で最大評価
      } else if (turnRatio >= 0.5) {
        baseValue = 0.7;  // 中盤で評価を上げる
      } else if (turnRatio >= 0.3) {
        baseValue = 0.5;  // 序盤後半から評価
      }
      break;
  }
  
  // 位置による価値（重要な領域を守る）
  final positionValue = _getEntanglementPositionValue(piece.position, gameState);
  
  // エンタングルペアの戦略的価値
  final strategicValue = _getEntanglementStrategicValue(
    piece,
    gameState,
    myColor,
    difficulty,
  );
  
  return baseValue + positionValue + strategicValue;
}

// エンタングルの位置による価値評価
double _getEntanglementPositionValue(Position position, GameState gameState) {
  // 中央付近のエンタングルはより価値が高い
  final centerDistance = _getDistanceFromCenter(position);
  final centerValue = (1.0 - centerDistance * 0.2).clamp(0.0, 1.0);
  
  // 相手の重要な領域（自分の駒が多い領域）を守る
  final defensiveValue = _evaluateDefensivePosition(position, gameState);
  
  return centerValue * 0.3 + defensiveValue * 0.2;
}

// 防御位置の評価
double _evaluateDefensivePosition(Position position, GameState gameState) {
  // この位置周辺に自分の重要な駒がある場合、防御価値が高い
  final board = gameState.board;
  var nearbyMyPieces = 0;
  
  // 周辺8マスをチェック
  for (int dr = -1; dr <= 1; dr++) {
    for (int dc = -1; dc <= 1; dc++) {
      if (dr == 0 && dc == 0) continue;
      final r = position.row + dr;
      final c = position.col + dc;
      if (r >= 0 && r < 8 && c >= 0 && c < 8) {
        final piece = board.getPiece(r, c);
        if (_isMyColor(piece, myColor)) {
          nearbyMyPieces++;
        }
      }
    }
  }
  
  return (nearbyMyPieces * 0.1).clamp(0.0, 0.5);
}

// エンタングルの戦略的価値評価
double _getEntanglementStrategicValue(
  Piece piece,
  GameState gameState,
  PlayerColor myColor,
  AIDifficulty difficulty,
) {
  if (difficulty != AIDifficulty.advanced) {
    return 0.0;  // 上級のみ戦略的評価
  }
  
  var strategicValue = 0.0;
  
  // 1. 相手のゲート適用を防御する価値
  // エンタングルされた駒はゲート適用を阻止する
  final blockingValue = _evaluateGateBlockingValue(piece, gameState, myColor);
  strategicValue += blockingValue;
  
  // 2. エンタングルペアの配置による価値
  // 重要な領域を複数のエンタングルで守る
  final pairValue = _evaluateEntanglementPairValue(piece, gameState, myColor);
  strategicValue += pairValue;
  
  // 3. 測定時の期待値
  // エンタングル状態は測定時に有利になる可能性がある
  final measurementValue = _evaluateEntanglementMeasurementValue(
    piece,
    gameState,
    myColor,
  );
  strategicValue += measurementValue;
  
  return strategicValue;
}

// ゲート適用の防御価値
double _evaluateGateBlockingValue(
  Piece piece,
  GameState gameState,
  PlayerColor myColor,
) {
  // エンタングルされた駒は、行/列のゲート適用を阻止する
  // 4マス選択のゲート適用も部分的に阻止する
  
  var blockingValue = 0.0;
  final position = piece.position;
  
  // 行/列のゲート適用を阻止する価値
  // この行/列に自分の重要な駒がある場合、防御価値が高い
  final rowValue = _evaluateRowDefense(position.row, gameState, myColor);
  final colValue = _evaluateColumnDefense(position.col, gameState, myColor);
  blockingValue += (rowValue + colValue) * 0.3;
  
  // 4マス選択のゲート適用を阻止する価値
  final areaValue = _evaluateAreaDefense(position, gameState, myColor);
  blockingValue += areaValue * 0.2;
  
  return blockingValue;
}

// 行の防御価値
double _evaluateRowDefense(
  int row,
  GameState gameState,
  PlayerColor myColor,
) {
  final board = gameState.board;
  var myPiecesInRow = 0;
  var importantPiecesInRow = 0;
  
  for (int col = 0; col < 8; col++) {
    final piece = board.getPiece(row, col);
    if (_isMyColor(piece, myColor)) {
      myPiecesInRow++;
      // 確定状態の駒はより重要
      if (piece.type == PieceType.white || piece.type == PieceType.black) {
        importantPiecesInRow++;
      }
    }
  }
  
  // 自分の駒が多い行を守る価値が高い
  return (myPiecesInRow * 0.1 + importantPiecesInRow * 0.2).clamp(0.0, 1.0);
}

// 列の防御価値
double _evaluateColumnDefense(
  int col,
  GameState gameState,
  PlayerColor myColor,
) {
  final board = gameState.board;
  var myPiecesInCol = 0;
  var importantPiecesInCol = 0;
  
  for (int row = 0; row < 8; row++) {
    final piece = board.getPiece(row, col);
    if (_isMyColor(piece, myColor)) {
      myPiecesInCol++;
      if (piece.type == PieceType.white || piece.type == PieceType.black) {
        importantPiecesInCol++;
      }
    }
  }
  
  return (myPiecesInCol * 0.1 + importantPiecesInCol * 0.2).clamp(0.0, 1.0);
}

// 領域の防御価値
double _evaluateAreaDefense(
  Position position,
  GameState gameState,
  PlayerColor myColor,
) {
  // この位置を含む4マス領域に自分の重要な駒がある場合、防御価値が高い
  final board = gameState.board;
  var myPiecesInArea = 0;
  
  // この位置を含む4マス領域をチェック
  for (int dr = 0; dr < 2; dr++) {
    for (int dc = 0; dc < 2; dc++) {
      final r = position.row + dr;
      final c = position.col + dc;
      if (r < 8 && c < 8) {
        final piece = board.getPiece(r, c);
        if (_isMyColor(piece, myColor)) {
          myPiecesInArea++;
        }
      }
    }
  }
  
  return (myPiecesInArea * 0.15).clamp(0.0, 0.6);
}

// エンタングルペアの価値
double _evaluateEntanglementPairValue(
  Piece piece,
  GameState gameState,
  PlayerColor myColor,
) {
  // エンタングルペアのもう一方の位置を取得
  final pairId = piece.entangledPairId;
  if (pairId == null) return 0.0;
  
  final pair = gameState.getEntangledPair(pairId);
  if (pair == null) return 0.0;
  
  // 2つのエンタングルが重要な領域を守っている場合、価値が高い
  final position1Value = _getEntanglementPositionValue(piece.position, gameState);
  final position2Value = _getEntanglementPositionValue(pair.otherPosition, gameState);
  
  // 2つのエンタングルが近い位置にある場合、連携した防御が可能
  final distance = _getDistance(piece.position, pair.otherPosition);
  final synergyValue = (1.0 - distance * 0.1).clamp(0.0, 1.0) * 0.2;
  
  return (position1Value + position2Value) * 0.3 + synergyValue;
}

// 測定時の期待値
double _evaluateEntanglementMeasurementValue(
  Piece piece,
  GameState gameState,
  PlayerColor myColor,
) {
  // エンタングル状態は測定時に50%で自分の色になる
  // ただし、エンタングルペアのもう一方も考慮する必要がある
  final pairId = piece.entangledPairId;
  if (pairId == null) return 0.0;
  
  // 測定時の期待値を計算
  // エンタングル状態は測定時に有利になる可能性がある
  return 0.1;  // 測定時の期待値（簡略化）
}

// 禁止領域の評価
double _evaluateForbiddenAreas(GameState gameState, PlayerColor myColor) {
  // 相手の禁止領域が多いほど、相手の選択肢が減る
  final opponentForbiddenAreas = gameState.getForbiddenAreas(
    _getOpponentColor(myColor)
  );
  return opponentForbiddenAreas.length * 0.1;
}
```

**3. 上級AI（高度な戦略）の実装**:

```dart
AIAction _thinkAdvanced(GameState gameState, List<GateType> availableGates) {
  // ミニマックスアルゴリズムまたはモンテカルロ木探索を使用
  // ここでは簡略化した評価関数を使用
  
  final candidates = <AIAction>[];
  
  for (final gate in availableGates) {
    final targets = _getPossibleTargets(gameState, gate);
    
    for (final target in targets) {
      // 仮想的にゲートを適用
      final simulatedState = _simulateGateApplication(
        gameState,
        gate,
        target,
      );
      
      // 測定後の予測確率を計算
      final measurementScore = _predictMeasurementScore(
        simulatedState,
        myColor,
      );
      
      // 相手の次の手を考慮（1手先読み）
      final opponentResponseScore = _evaluateOpponentResponse(
        simulatedState,
        myColor,
      );
      
      // 盤面評価（上級AIの難易度を考慮）
      final boardEvaluation = _evaluateBoard(
        simulatedState,
        myColor,
        AIDifficulty.advanced,  // 上級AIの難易度を渡す
      );
      
      // 総合評価（測定予測 + 相手の応手考慮 + 盤面評価）
      final totalScore = measurementScore * 0.4 
          - opponentResponseScore * 0.3 
          + boardEvaluation.totalScore * 0.3;
      
      candidates.add(AIAction(
        gate: gate,
        target: target.target,
        positions: target.positions,
        score: totalScore,
      ));
    }
  }
  
  // 最適な行動を選択
  candidates.sort((a, b) => b.score.compareTo(a.score));
  return candidates.first;
}

// 測定後の予測スコア
double _predictMeasurementScore(
  GameState gameState,
  PlayerColor myColor,
) {
  var myExpectedPieces = 0.0;
  var opponentExpectedPieces = 0.0;
  
  final board = gameState.board;
  
  for (int row = 0; row < 8; row++) {
    for (int col = 0; col < 8; col++) {
      final piece = board.getPiece(row, col);
      
      // 測定後の期待値を計算
      final (myProb, opponentProb) = _getMeasurementProbability(
        piece,
        myColor,
      );
      
      myExpectedPieces += myProb;
      opponentExpectedPieces += opponentProb;
    }
  }
  
  // 自分の駒が多いほど良い
  return myExpectedPieces - opponentExpectedPieces;
}

// 測定確率の計算
(double, double) _getMeasurementProbability(
  Piece piece,
  PlayerColor myColor,
) {
  switch (piece.type) {
    case PieceType.white:
      return myColor == PlayerColor.white ? (1.0, 0.0) : (0.0, 1.0);
    case PieceType.black:
      return myColor == PlayerColor.black ? (1.0, 0.0) : (0.0, 1.0);
    case PieceType.grayPlus:
    case PieceType.grayMinus:
      // 50%で白、50%で黒
      return (0.5, 0.5);
    case PieceType.blackWhite:
    case PieceType.whiteBlack:
      // エンタングル状態も50%で白、50%で黒
      return (0.5, 0.5);
  }
}

// 相手の応手を評価
double _evaluateOpponentResponse(
  GameState gameState,
  PlayerColor myColor,
) {
  // 相手が最適な手を打った場合の評価
  // 簡略化: 相手の使用可能なゲートで最悪の結果を想定
  final opponentAvailableGates = _getAvailableGatesForOpponent(gameState, myColor);
  
  if (opponentAvailableGates.isEmpty) {
    return 0.0;  // 相手が打てない場合は問題なし
  }
  
  var worstScore = double.infinity;
  
  for (final gate in opponentAvailableGates) {
    final targets = _getPossibleTargetsForOpponent(gameState, gate, myColor);
    
    for (final target in targets) {
      final simulatedState = _simulateGateApplication(
        gameState,
        gate,
        target,
      );
      
      final evaluation = _evaluateBoard(simulatedState, myColor);
      worstScore = math.min(worstScore, evaluation.totalScore);
    }
  }
  
  return worstScore == double.infinity ? 0.0 : worstScore;
}
```

```dart
// 使用可能なゲートを取得
List<GateType> _getAvailableGates(GameState gameState) {
  final myCooldowns = gameState.getCooldowns(myColor);
  final availableGates = <GateType>[];
  
  for (final gate in GateType.values) {
    final cooldown = myCooldowns[gate] ?? 0;
    if (cooldown == 0) {
      availableGates.add(gate);
    }
  }
  
  return availableGates;
}

// 可能な適用領域を取得
List<TargetSelection> _getPossibleTargets(
  GameState gameState,
  GateType gate,
) {
  if (_isTwoBitGate(gate)) {
    return _getPossibleAdjacentPairs(gameState);
  } else {
    final targets = <TargetSelection>[];
    
    // 行/列の選択
    targets.addAll(_getPossibleRows(gameState));
    targets.addAll(_getPossibleColumns(gameState));
    
    // 4マスの選択
    targets.addAll(_getPossibleFourPieces(gameState));
    
    return targets;
  }
}

// ゲート適用のシミュレーション
GameState _simulateGateApplication(
  GameState gameState,
  GateType gate,
  TargetSelection target,
) {
  // ゲーム状態をコピー
  final simulatedState = gameState.copy();
  
  // ゲートを適用
  simulatedState.applyGate(gate, target);
  
  return simulatedState;
}

// 2ビットゲートかどうか
bool _isTwoBitGate(GateType gate) {
  return gate == GateType.cnot || gate == GateType.swap;
```

**5. 難易度別のパラメータ調整**:

```dart
class AIParameters {
  // 評価関数の重み
  final double measurementWeight;  // 測定予測の重み
  final double opponentResponseWeight;  // 相手応手の重み
  final double boardEvaluationWeight;  // 盤面評価の重み
  final double randomness;  // ランダム性（0.0=完全最適、1.0=完全ランダム）
  final int lookAheadDepth;  // 先読みの深さ
  
  static AIParameters getBeginner() {
    return AIParameters(
      measurementWeight: 0.0,
      opponentResponseWeight: 0.0,
      boardEvaluationWeight: 0.0,
      randomness: 1.0,
      lookAheadDepth: 0,
    );
  }
  
  static AIParameters getIntermediate() {
    return AIParameters(
      measurementWeight: 0.0,
      opponentResponseWeight: 0.0,
      boardEvaluationWeight: 1.0,
      randomness: 0.3,  // 上位3つから選択
      lookAheadDepth: 0,
    );
  }
  
  static AIParameters getAdvanced() {
    return AIParameters(
      measurementWeight: 0.4,
      opponentResponseWeight: 0.3,
      boardEvaluationWeight: 0.3,
```

**ガイドシステムの実装**:

**ガイド状態管理**:
- `currentStep`: 現在のステップ番号（1-7）
- `stepCompleted`: 各ステップの完了状態
- `guideMode`: ガイドモードの有効/無効

- 正しい操作時は次のステップに進む

**状態確率分布の管理**:
- 8通りの状態すべての確率振幅を保持: `amplitudes[8]`（複素数配列）
- 各状態の確率: `probabilities[i] = |amplitudes[i]|²`
- 確率の合計が1になることを常に確認（正規化）

**アルゴリズムステップ（実装詳細）**:

1. **初期化フェーズ**:
   - **実装方法**: 
     - 各マス（1-3マス）に順次Hゲートを適用
     - または全マスに一括でHゲートを適用
   - **状態管理**: 
     - 各マスの状態を|+⟩（グレープラス）に設定
     - 確率分布を均一に初期化: P(i) = 1/16 = 6.25% for all i (i=0..15)
     - 各状態の確率振幅: amplitudes[i] = 1/4 for all i
   - **検証**: 全4マスが|+⟩状態であることを確認
   - **確率分布表示**: 右側パネルで16状態すべてが6.25%であることを表示

2. **オラクル条件の自動設定**:
   - **システムによる自動設定**:
     - システムが自動的に探す状態を決定（例: |1010⟩、または白が2個の状態等）
     - 条件を満たす状態のリストを作成（1つまたは複数）
   - **条件タイプの例**:
     - **特定の状態**: 例: |1010⟩を探す
     - **パターン条件**: 例: 白が2個の状態（|0011⟩, |0101⟩, |0110⟩, |1001⟩, |1010⟩, |1100⟩）
   - **条件評価**: 
     - 全16状態に対して条件を評価
     - 条件を満たす状態のインデックスリストを作成
     - 右側の確率分布パネルで視覚的にハイライト表示

3. **オラクルゲート適用（位相反転）**:
   - **実装方法**: 
     - 条件を満たす状態の確率振幅に-1を掛ける
     - 専用の「オラクルゲート適用」ボタンで一括適用
   - **位相反転の計算**: 
     - for each state i where condition(i):
       - amplitudes[i] = -amplitudes[i]
     - 確率は |振幅|² なので、符号の反転だけでは確率は変わらない
   - **数式**: O|x⟩ = -|x⟩ (if condition(x)), |x⟩ (otherwise)
   - **確率分布表示**: 
     - 右側パネルで、条件を満たす状態の確率振幅が反転することを表示
     - 確率は変わらないが、振幅の符号が変わることを説明

4. **拡散演算子適用（振幅増幅）**:
   - **実装方法**: 
     - 各マスにHゲートを適用（重ね合わせ状態に戻す）
     - 平均値周りの反転を計算: D = 2|s⟩⟨s| - I
     - 条件を満たす状態の確率振幅を増幅
   - **拡散演算子の計算**:
     - 平均振幅: avg = (1/16) × Σ amplitudes[i]
     - 各状態の振幅を更新: amplitudes[i] = 2×avg - amplitudes[i]
     - これにより、平均より上にある振幅は増加し、平均より下にある振幅は減少
   - **確率分布表示**: 
     - 右側パネルで、条件を満たす状態の確率が増加することを表示
     - 成功確率を計算: P_success = Σ |amplitudes[i]|² for i where condition(i)

5. **繰り返し（オラクル + 拡散演算子）**:
   - **実装方法**: 
     - ステップ3と4を繰り返し実行
     - 最適繰り返し回数: k ≈ π/4 × √N ≈ 3 (N=16)
   - **各繰り返しでの処理**:
     - オラクルゲート適用（位相反転）
     - 拡散演算子適用（振幅増幅）
     - 成功確率の更新: P_success(k) = sin²((2k+1)θ) where sin(θ) = 1/4
   - **確率分布表示**: 
     - 右側パネルで、繰り返しごとの成功確率を表示
     - 各状態の確率が更新されることを表示
     - 最適回数に達したら通知

6. **測定フェーズ**:
   - **実装方法**: 
     - 全4マスに対して測定操作を実行
     - 各状態の確率に基づいて確率的に1つの状態を選択
     - 選択された状態に基づいて、各マスを白/黒に確定
   - **測定の計算**:
     - 確率分布に基づいてランダムに1つの状態を選択
     - 選択された状態（例: |1010⟩）に基づいて、各マスを確定
       - マス1: 1 → 黒
       - マス2: 0 → 白
       - マス3: 1 → 黒
       - マス4: 0 → 白
   - **結果判定**: 
     - 測定後の状態が条件を満たすかチェック
     - 成功/失敗を表示
     - 成功確率の最終結果を表示
   - **確率分布表示**: 
     - 右側パネルで、選択された状態が100%に確定することを表示
     - 他の状態は0%になる

**実装上の制約**:
- 1×4盤面（4マス）を使用し、16通りの状態のみを扱います
- 各マスを独立した量子ビットとして扱います（完全なエンタングルメントは限定的）

## チュートリアル概要
ゲームモード選択画面の一番上に「チュートリアル」ボタンを配置し、ゲームの遊び方と量子コンピュータの基本概念を説明するモード。
全ページに画像またはアニメーションを配置し、視覚的に理解しやすくする。
インタラクティブデモは不要（チャレンジモード・フリーランモードで体験可能）。

### スワイプ分割について
説明文が多いページは、無理に1枚に収めず、スワイプで2-3枚に分割可能。
- 各ページ内で複数のスライドに分割（例: ページ3-1、ページ3-2、ページ3-3）
- スワイプ（左右スワイプ）で次のスライド/前のスライドに移動
- 進捗表示: 「ページ番号-スライド番号/総スライド数」（例: 3-1/3-3）
- 各スライドに「前へ」「次へ」ボタンを配置
- 最後のスライドで「次のページへ」ボタンを表示
- ページ全体の進捗も表示（例: ページ3/14）

## 実装上の注意事項

1. **アニメーションの統一**: 各ゲート説明ページで、変換アニメーションの速度とスタイルを統一する
2. **色の統一**: ゲートアイコンの色はゲーム内の実装と一致させる
3. **駒の表示**: ゲーム内の実装と同じスタイルで駒を表示する
4. **ナビゲーション**: 
   - 各ページに「前へ」「次へ」ボタンを配置
   - スワイプ（左右スワイプ）でも移動可能
   - 最後のスライドで「次へ」ボタンをアクティブ表示
5. **進捗表示**: 
   - ページ全体の進捗: ページ番号（例: ページ3/14）
   - ページ内のスライド進捗: 下にスワイプで進行するドットマークで表現
6. **再表示機能**: ゲームモード選択画面からいつでもチュートリアルを再開できる
7. **スワイプ分割の実装**:
   - 説明文が多いページは、自然な区切りで2-3スライドに分割
   - 各スライドは独立した画面として実装
   - スライド間の遷移はスムーズなアニメーション（フェードまたはスライド）
   - スライドの分割は、テキストの内容と視覚要素の関係を考慮して決定
